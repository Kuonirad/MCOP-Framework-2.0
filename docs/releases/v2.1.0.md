# v2.1.0 — Dialectical Hardening & Release Automation

**Tag target:** `c1dfcf3` (2026-04-19, HEAD of `main` at release time)

Minor release. The triad kernels get meaningful performance work, the
maintenance story is formalized with new workflows and governance, and the
release process moves onto rails with `release-drafter`.

## 🚀 Features

- `release-drafter` workflow and config for automated, label-driven release
  notes on every merge to `main`.
- Scheduled `stale` workflow to keep the PR/issue backlog from regressing.
- `auto-close-bot-prs` workflow that daily closes duplicate automated PRs and
  cleans up their branches.
- `delete-merged-branches` workflow that removes PR branches on merge.
- `GOVERNANCE.md` codifying the maintainer roster, lazy-consensus decision
  model, release cadence, and the security-escalation path.
- `.github/FUNDING.yml` for GitHub Sponsors.
- Dedicated `docs/releases/` directory with versioned notes.

## ⚡ Performance

- NovaNeoEncoder `estimateEntropy` rewritten with native `for` loops instead
  of `Array.prototype.reduce` + `Math.pow`. Measurable throughput gain on
  large contexts.
- StigmergyV5 resonance search caches vector magnitudes, yielding an
  effectively O(N) resonance query where the previous path was O(N²) in the
  dot-product denominator.
- Pino logging overhead reduced by moving payload evaluation behind level
  checks.

## 🛡️ Security

- Pino logger hardened with an explicit `redact` array that suppresses known
  sensitive payloads from structured logs.
- Sensitive-data-exposure CodeQL alert in CLI utilities fixed.

## 🐛 Fixes

- `actions/upload-artifact` pinned to `@v4`.
- ESLint configuration reconciled with Next.js 16.
- Accessibility: decorative icons (`File`, `Window`, `Globe`, Vercel) marked
  `aria-hidden` or given empty alt text.

## 🧰 Maintenance

- README badges replaced with CI / CodeQL / Releases / License / Contributors
  / Maintained. Obsolete eco-fitness and bus-factor badges removed.
- `.all-contributorsrc` expanded to reflect the real author roster.
- `ROADMAP_TO_100.md` marked historical; operative roadmap now lives in
  `GOVERNANCE.md` + `CHANGELOG.md`.
