# v2.0.2 — Security Hygiene & CI Hardening

**Tag target:** `d47cbbe` (2026-04-09)

Second patch release. Concentrates on security posture and a full CI migration
from npm to pnpm with SHA-pinned actions.

## Added

- Cryptographically strong (`crypto.randomUUID`) trace identifiers in
  StigmergyV5 to eliminate collision risk.
- Container health-check endpoint (`/api/health`) for Docker orchestration.
- Comprehensive CSP / HSTS / frame-options security headers.

## Changed

- CI pipeline migrated from npm to pnpm across all workflows.
- All workflow actions pinned to SHA-addressed versions.
- Composite CI setup action extracted; `checkout` hoisted to workflow level
  to resolve a composite-action loading failure.
- Docker base image pinned to a specific digest.
- Trojan-source guard, malicious-mod scanner, and CodeQL placed on a regular
  cadence.

## Fixed

- CodeQL sensitive-data-exposure alert in CLI logging path.
- `publish.yml` syntax error that blocked the release job.
- `eco-fitness` math-utility error producing NaN for empty traces.
- `actions/upload-artifact` pinned to `@v4` (previous reference used an invalid
  SHA).

## Security

No CVEs of record. Hardening-only release.
