# v2.2.0 тАФ Supply-Chain Hardening, Public Benchmarks, Coverage Push

**Release date:** 2026-04-30
**Tag:** `v2.2.0` (framework) ┬╖ `npm-v0.2.0` (`@kullailabs/mcop-core`) ┬╖ `py-v3.2.0` (`mcop`)

This release closes Phases 1 & 2 of the [post-audit roadmap](../audits/independent-audit-response-2026-04-30.md). Releases now ship with attached, schema-validated CycloneDX SBOMs; the deterministic-triad benchmarks have a public dashboard at `/benchmarks`; an end-to-end ONNX `IEmbeddingBackend` example is in-tree; and project-wide branch coverage moved 81.96% тЖТ 84.57%.

## ЁЯЪА Features

- **CycloneDX SBOMs auto-attached to every release.** Both `publish-npm.yml`
  and `publish-pypi.yml` now run `pnpm sbom` + `pnpm sbom:validate` before
  publishing and attach `mcop-framework.cdx.json` + `mcop-core.cdx.json` to
  the GitHub Release via `softprops/action-gh-release@v2.6.2`.
- **Schema validator.** `pnpm sbom:validate` validates each generated SBOM
  against the official CycloneDX JSON schemas (1.0тАУ1.7 supported via
  `@cyclonedx/cyclonedx-library`). Non-conformant SBOMs abort the release.
- **Public benchmarks dashboard.** `/benchmarks` renders the deterministic
  Human-vs-Pure-AI-vs-MCOP fixture from `docs/benchmarks/results.json`
  (regenerable via `pnpm benchmark:refresh`).
- **TypeDoc API docs** for `@kullailabs/mcop-core` at `docs/api/core/`,
  generatable via `pnpm typedoc`.
- **Self-contained coverage badge** at `docs/badges/coverage.svg` (94.24%
  yellow-green at release time), refreshed by `pnpm coverage:badge`.
- **ONNX `IEmbeddingBackend` example** at `examples/onnx_embedding_backend.ts`
  тАФ runnable `all-MiniLM-L6-v2` reference plus dynamic-import fallback so the
  file typechecks without `onnxruntime-node` installed.
- **NOVA-NEO embedding backend.** `NovaNeoConfig.backend = 'embedding'` opts
  into n-gram feature-hashing for semantically-correlated activations
  (zero-dependency, deterministic, cross-platform). `'hash'` remains the
  byte-identical v1.x default.

## ЁЯЫбя╕П Supply Chain & Provenance

- Both publish workflows run `pnpm sbom:validate` against every SBOM before
  any artefact is uploaded тАФ a malformed SBOM aborts the release.
- `permissions.contents: write` is scoped per-job (publish only), keeping
  the build job's token blast radius minimal.
- npm publish continues to use OIDC Trusted Publishing + automatic Sigstore
  provenance; the SBOM steps run alongside, not in place of, those.

## ЁЯПЧя╕П Workspace & Tooling
