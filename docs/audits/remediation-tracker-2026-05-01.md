## Audit Errata (factual corrections)

The master audit report contained a small number of factual inaccuracies
relative to the repository's actual state on the receipt date. They are
recorded here for completeness:

1. **"Dependabot.yml not explicitly confirmed in tree"** — file is present
   at [`.github/dependabot.yml`](../../.github/dependabot.yml) and has been
   for multiple releases.
2. **"`GOVERNANCE.md` not prominently linked in top-level README"** — it is
   linked twice in [`README.md`](../../README.md) (badge on line 12, prose
   link on line 181).
3. **"No public code-coverage badge"** — present and rendered in the README
   header (line 5), backed by Codecov upload from CI.
4. **"No matrix testing across Node/Python versions visible"** — the main
   workflow [`.github/workflows/ci.yml`](../../.github/workflows/ci.yml)
   runs on Node 20.x + 22.x and Python 3.10 + 3.12.
5. **"Full Sigstore attestation not attached to every release note"** —
   npm trusted publishing automatically generates Sigstore provenance for
   every release; PyPI uses OIDC trusted publishing.

These errata do not change the auditor's overall headline score; they
narrow the genuine remaining-work surface to mutation / fuzz testing,
OpenTelemetry, i18n, hosted TypeDoc deployment, and (now closed in this
PR) Lighthouse CI.
