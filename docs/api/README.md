# MCOP Framework — API documentation

API reference docs are generated by [TypeDoc](https://typedoc.org/) from
the TypeScript source of the published `@kullailabs/mcop-core` library.
Output is gitignored (`docs/api/core/` is regenerated on demand) so the
reference HTML never drifts out of sync with the code.

## Generate locally

```bash
pnpm typedoc           # convenience root-level alias
# or
pnpm --filter @kullailabs/mcop-core typedoc
```

The generator writes static HTML + assets to `docs/api/core/`. Open
`docs/api/core/index.html` in a browser, or serve the directory with any
static server:

```bash
npx --yes serve docs/api/core
```

## Configuration

`packages/core/typedoc.json` controls entry points, output path,
exclusions, and the navigation links rendered into every page. Edit
that file to tune what shows up in the generated docs.

## Publishing

The maintainer wires the `pnpm typedoc` step into a publish/release
workflow (e.g. GitHub Pages) — out of scope for the initial bootstrap
because workflow YAMLs require a token scope this repo's automation
does not currently grant. A minimal Pages workflow would look like:

```yaml
- name: Generate API docs
  run: pnpm typedoc
- name: Deploy to GitHub Pages
  uses: actions/deploy-pages@v4
  with:
    folder: docs/api/core
```
